# Formulon

> **ステータス: 開発中。本番利用にはまだ適しません。**
> 最初のタグ付きリリースまで、API・ファイル構成・パッケージングは予告なく変更される可能性があります。

Formulon は **ヘッドレス Excel** を目指す C++17 製の計算エンジンです。Excel
をインストールすることなく、Excel 365 の数式を評価し、`.xlsx` ワークブック
を読み書きすることを目的としています。同一のエンジンをブラウザ
(WebAssembly)・Python・ネイティブ CLI 向けにパッケージ配布し、どの環境でも
ワークブックを同じ結果で再計算できることを目指します。

## ゴール

- **Excel 365 の忠実な再現。** Mac Excel 365 (ja-JP ロケール) をリファレンス
  オラクルとして扱います。意図的な差分は隠さず明示的に記録します。
- **ヘッドレスで埋め込み可能。** Excel のインストール・GUI・COM オートメー
  ション不要。ライブラリとしてリンク／インポート／ロードできます。
- **一つのエンジン、複数の配布形態。** 単一の C++ コアを以下の形で提供
  します:
  - **WebAssembly** モジュール (npm)。ブラウザおよび Node.js 向け。
  - **Python** wheel (PyPI)。
  - ネイティブ **CLI** バイナリ (Linux / macOS / Windows)。
- **小さい WASM フットプリント。** WebAssembly ビルドは厳しいサイズ予算を
  守り、Web アプリに組み込んでもペイロードを圧迫しないことを目指します。
- **読みやすく、レビュー可能なコード。** 構造化されたエラーハンドリング、
  RAII、例外を使わない設計、限定された依存関係により、規模が大きくなっても
  監査可能な状態を維持します。

## 想定ユースケース

Excel を起動せずにスプレッドシートを計算したい場面:

- バッチジョブやデータパイプラインで `.xlsx` をヘッドレスに再計算する
- Web アプリ内 (ブラウザ) で Excel 風の数式を評価する
- 社内ツール・ボット・ノートブックに計算機能を埋め込む
- 数式の検証やレガシースプレッドシートの移行

## スコープ

Formulon は計算エンジンとファイル I/O にフォーカスします。スプレッドシート
UI そのものの提供は目的としていません。UI を構築しやすくするための軽量な
統合レイヤー (数式バー、グリッド連携) は別途計画しています。
